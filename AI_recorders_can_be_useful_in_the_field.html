📖 Why was this built? / 왜 이렇게 만들었는가?

⚠ FIELD SAFETY — READ FIRST

For everyday personal use, installing this as a PWA on the home screen is acceptable.

For field use in conflict zones, refugee camps, or under hostile state surveillance, do NOT install this as a PWA. The home-screen icon itself becomes a piece of evidence on a seized device. Use the page directly through a browser tab that can be closed and history-cleared, or load it from an offline file via local storage.

Calculator camouflage protects the running app — not the icon. A "Calculator" PWA icon on the home screen is unusual enough to trigger inspection by trained personnel.

Purpose of this build

This app is a review-stage demonstrator for OTF Security Lab (#21441). It is not for immediate field deployment. Its purpose is to validate why such a design is needed in humanitarian contexts.

1. Why a single HTML file?

VitalGuard's core philosophy: "auditable single file = supply chain attack immunity". Zero external libraries, CDNs, or API calls — only one SHA-256 hash needs verification. Approximately 100–250× lighter than equivalent library-dependent code.

Service Worker and Manifest are inlined via data: URLs to preserve single-file integrity.

2. Why no password at recording time?

Earlier versions required a password before each recording. Field testing (and reviewer feedback) showed this fails the most basic field reality: in an emergency, every second of fumbling with a password creates risk.

This version removes the password barrier entirely. Recording starts the instant the user taps. Encryption can be added at the storage layer in production builds without blocking the capture path.

3. Why scientific calculator camouflage?

A basic calculator on a frontline worker's phone is mildly suspicious — most people use the OS calculator. A scientific calculator is plausible for engineers, students, surveyors, medical personnel, and field researchers. It also offers more "buttons" to hide secret triggers in plain sight.

Secret trigger: 1234 × 5678 = enters recording mode. Settings unlock: 0000 on the display restores the ⚙ icon when hidden.

4. Why hideable settings?

An always-visible ⚙ icon on a calculator app is itself suspicious. Tapping OFF in Settings removes the icon entirely; the user becomes a person with "just a calculator." To restore settings, the user types the unlock code (default 0000) on the calculator display.

This pattern — secret keystroke unlocks hidden function — is well-established in counter-surveillance UX and minimizes attack surface during a casual device inspection.

5. Why an instant-recording mode?

Sudden violence or evidence-capture moments tolerate no 1–2 second delay. With instant mode enabled, recording starts in the background the moment the PWA launches, while the screen displays a normal calculator.

6. Why automatic acoustic event recording?

Gunshots, explosions, and screams produce clear amplitude spikes (100–120 dB). The Web Audio API's AnalyserNode is used to compute RMS energy, Zero-Crossing Rate, and Spectral Flatness; recording auto-starts when thresholds are exceeded. Implemented in pure DSP without external ML libraries.

This means evidence is preserved automatically even when the user is incapacitated.

7. Why steganographic filenames?

Names like meeting_1735892341.webm reveal intent on a quick gallery scan. Saved as IMG_20260105_184523.jpg instead, the file appears as a "broken photo" that inspectors typically ignore.

8. Why automatic chunk splitting?

A single 30-minute file exposes everything if seized. Splitting every 30 seconds limits exposure per file. Chunks are saved at randomized intervals to obscure temporal ordering.

9. Why panic wipe (3-tap)?

A visible "delete" button signals "something is here." Three rapid taps within 1 second on an invisible top-left zone permanently destroys all recordings, settings, and cache — with no visible UI change.

This directly implements Morgan J. Ethical Principle #10 — "Freedom of data deletion."

10. Why metadata stripping?

WebM containers embed encoder identifiers ("Chrome 120 on Android"), exact timestamps, and codec fingerprints — usable for device tracking and time-zone inference. These are randomized before storage to harden activist protection.

11. Why GDPR · UNCRC · CRPD alignment by design?

This app guarantees, from the design stage:

  • GDPR: zero servers → zero data transfer → zero processor liability
  • UNCRC: child-protection alerts without any data collection
  • CRPD: simple UI usable by non-specialists

Compliance is not "added later" — it is structurally impossible to violate.

Note to OTF reviewers

This is a pre-deployment demonstrator. Production deployment will add:

  • Independent security audit (ROS — Radically Open Security)

분쟁지역·난민캠프·적대적 국가 감시 환경에서는 절대 PWA로 설치하지 마세요. 홈화면의 아이콘 자체가 압수 시 증거물이 됩니다. 브라우저 탭으로 직접 열어 사용하고 닫은 후 기록을 지우거나, 로컬 파일에서 직접 로드하는 방식을 사용하세요.

계산기 위장은 실행 중인 앱을 숨길 뿐 아이콘 자체는 숨기지 못합니다. 홈화면의 "Calculator" PWA 아이콘은 훈련된 검사관에게는 충분히 의심스러운 표적입니다.

이 앱의 본질

이 앱은 OTF Security Lab 심사용 (#21441) 데모입니다. 즉시 현장 배포용이 아니라, "왜 이런 설계가 인도주의 환경에서 필요한가"를 검증하기 위한 참고 구현체입니다.

1. 왜 단일 HTML 파일인가?

VitalGuard의 핵심 철학은 "감사 가능한 단일 파일 = 공급망 공격 면역"입니다. 외부 라이브러리·CDN·API가 0개이므로, 파일 1개의 SHA-256 해시만 검증하면 됩니다. 표준 라이브러리 의존 코드 대비 약 100~250배 경량입니다.

Service Worker와 Manifest는 모두 data: URL로 인라인 처리하여 단일 파일 무결성을 보장합니다.

2. 왜 녹음 시 비밀번호가 없는가?

이전 버전은 매 녹음 시마다 비밀번호를 요구했습니다. 그러나 현장 테스트와 심사위원 피드백 결과, 가장 기본적인 현장 현실에 어긋남이 드러났습니다 — 긴급 상황에서 비밀번호를 입력하느라 보내는 1초 1초가 위험을 키웁니다.

이번 버전은 비밀번호 장벽을 완전히 제거했습니다. 사용자가 탭하는 즉시 녹음이 시작됩니다. 암호화는 캡처 경로를 막지 않으면서 저장 단계에서 추가할 수 있습니다.

3. 왜 공학용 계산기 위장인가?

일반 계산기는 일선 활동가의 폰에 있다는 것 자체가 다소 의심스러울 수 있습니다 — 대부분은 OS 기본 계산기를 씁니다. 반면 공학용 계산기는 엔지니어·학생·측량사·의료진·현장 연구자에게 자연스럽습니다. 더 많은 "버튼"을 통해 비밀 트리거를 평범한 자리에 숨길 수 있는 장점도 있습니다.

비밀 트리거: 1234 × 5678 = 입력 시 녹음 모드 진입. 설정 잠금 해제: 설정 숨김 상태에서 계산기에 0000 입력 시 ⚙ 아이콘 복구.

4. 왜 설정 숨김 기능인가?

계산기 앱에 항상 보이는 ⚙ 아이콘은 그 자체로 의심스럽습니다. 설정에서 OFF를 누르면 아이콘이 완전히 사라지며, 사용자는 "그냥 계산기를 가진 사람"이 됩니다. 설정을 복구하려면 계산기 화면에 잠금 해제 코드(기본 0000)를 입력합니다.

이 패턴 — 비밀 키 입력으로 숨겨진 기능 잠금 해제 — 은 반-감시 UX에서 검증된 방식이며, 일상적 디바이스 검사에서 공격 표면을 최소화합니다.

5. 왜 즉시 녹음 모드가 필요한가?

현장에서 갑작스러운 폭력 사건·증거 확보 상황은 1~2초의 지연도 치명적입니다. 즉시 녹음 모드를 켜두면 PWA 클릭과 동시에 백그라운드 녹음이 시작되며, 화면에는 평범한 계산기가 표시됩니다.

6. 왜 음향 이벤트 자동 녹음인가?

총성·폭발음·비명 같은 굉음은 100~120dB의 명확한 진폭 스파이크를 보입니다. Web Audio API의 AnalyserNode로 RMS 에너지·Zero-Crossing Rate·스펙트럼 평탄도(Spectral Flatness)를 분석하여 임계값 초과 시 자동 녹음합니다. 외부 ML 라이브러리 없이 순수 DSP로 구현되었습니다.

이는 사용자가 의식을 잃거나 행동할 수 없는 위급 상황에서도 증거가 자동 보존됨을 의미합니다.

7. 왜 스테가노 파일명인가?

meeting_1735892341.webm 같은 이름은 갤러리 검색 즉시 노출됩니다. 대신 IMG_20260105_184523.jpg 형식으로 저장하면 갤러리에서 "깨진 사진"으로 보여 검사관이 무시할 가능성이 높습니다.

8. 왜 자동 청크 분할인가?

30분짜리 단일 파일은 압수 시 모든 대화가 한 번에 노출됩니다. 30초마다 분할하면 한 파일이 발견되어도 30초만 노출됩니다. 청크는 무작위 시간 간격으로 저장되어 시간 순서 추론도 어렵게 합니다.

9. 왜 패닉 와이프(3-탭 삭제)인가?

"즉시 삭제" 버튼은 검사관에게 "여기 뭔가 있다"는 단서를 줍니다. 화면 좌상단 보이지 않는 영역을 1초 이내 3번 탭하면 모든 녹음·설정·캐시가 즉시 영구 삭제됩니다. 외관상 변화 없이 완료됩니다.

이는 Morgan J. 윤리 원칙 #10 — "데이터 삭제의 자유"의 직접 구현입니다.

10. 왜 메타데이터 제거인가?

WebM 컨테이너는 인코더 식별자(예: "Chrome 120 on Android"), 정확한 타임스탬프, 코덱 핑거프린트를 포함합니다. 이는 디바이스 추적·시간대 추정에 사용될 수 있습니다. 저장 직전 무작위화하여 활동가 보호를 강화합니다.

11. 왜 GDPR · UNCRC · CRPD 정렬인가?

이 앱은 설계 단계부터 다음을 보장합니다:

  • GDPR: 서버 0개 → 데이터 이전 없음 → 처리자 책임 0
  • UNCRC: 아동 보호 시 데이터 수집 없이 위험 알림만
  • CRPD: 단순 UI로 비전문가도 사용 가능

"규정 준수를 나중에 추가"가 아니라 "처음부터 위반이 불가능한 구조"입니다.
