📖 Why was this built? / 왜 이렇게 만들었는가?

⚠ FIELD SAFETY — READ FIRST

For everyday personal use, installing this as a PWA on the home screen is acceptable.

For field use in conflict zones, refugee camps, or under hostile state surveillance, do NOT install this as a PWA. The home-screen icon itself becomes evidence on a seized device. Use the page directly through a browser tab that can be closed and history-cleared, or load it from an offline file via local storage.

Calculator camouflage protects the running app — not the icon.

💾 About the Save Format option (V3.0)

An earlier version corrupted recordings by inserting random padding bytes into the file header. V3.0 fixes this with a clean two-option model:

  • As-is (default): The recording is saved unmodified. It plays directly in any audio player. Use this when you simply want to record evidence.
  • Image disguise: The recording is wrapped inside a JPEG envelope and saved with a .jpg extension. The gallery app will treat it as a "broken photo" and skip it. To play the audio, you must manually rename IMG_20260105_xxx.jpg back to .webm on a computer. The audio data itself is preserved unchanged inside the envelope, so renaming restores playback fully.

You can change this option at any time. It only affects future recordings, not files already saved.

The two-screen model

The app has two distinct screens:

  • Recorder screen: Normal UX. Toasts, dialogs, and the help button work here. Default on first launch.
  • Calculator screen: A working scientific calculator. The app is absolutely silent here — no toasts, no dialogs, no recording indicator, no help. Even when recording is active, the screen behaves like a normal calculator. This is the operational mode for risky environments.

Switch between them using the toggle at the top of the recorder screen.

Hidden gear icon

The ⚙ icon can be hidden so the app appears to be only a calculator. To restore the settings panel: switch to the calculator screen, type 0000 (or your changed code), then press =. The code is user-changeable.

If you tap "Hide" from the recorder screen, the app first switches to the calculator and then removes the gear icon, so you can recover via the keypad.

Why no password at recording time?

Earlier versions required a password before each recording. Field testing showed this fails the most basic field reality: in an emergency, every second of fumbling with a password creates risk. V3.0 removes the password barrier. Recording starts the instant the user taps. Encryption can be added at the storage layer in production builds without blocking the capture path.

Single HTML file, zero dependencies

VitalGuard's core philosophy: auditable single file = supply chain attack immunity. Zero external libraries, CDNs, or API calls — only one SHA-256 hash needs verification. Service Worker and Manifest are inlined via data: URLs.

Acoustic / keyword / BLE triggers

Gunshots, explosions, and screams produce clear amplitude spikes (100–120 dB). The Web Audio API's AnalyserNode computes RMS and Zero-Crossing Rate; recording auto-starts when thresholds are exceeded. Pure DSP — no external ML.

When triggered while the calculator screen is active, recording starts silently — no toast, no indicator. Evidence is preserved without exposing the user.

Panic wipe (3-tap)

A visible "delete" button signals "something is here." Three rapid taps within 1 second on the invisible top-left zone permanently destroy all recordings, settings, and cache — with no visible UI change. Works on both screens.

Implements Morgan J. Ethical Principle #10 — Freedom of data deletion.

GDPR · UNCRC · CRPD alignment by design

  • GDPR: zero servers → zero data transfer → zero processor liability
  • UNCRC: child-protection alerts without any data collection
  • CRPD: simple UI usable by non-specialists

Compliance is structural, not bolted on.

Pre-deployment status

This is a demonstrator. Production deployment will add independent security audit (ROS), multilingual UI (Arabic, Rohingya, etc.), field pilots (UCL GDI Hub, IDS Sussex), Hippocratic License 3.0 liability clauses, and optional storage-layer AES-256-GCM that does not block the capture path.

⚠ 현장 안전 — 먼저 읽으세요

일상 개인 용도로는 PWA 형태로 홈화면에 설치해도 괜찮습니다.

분쟁지역·난민캠프·적대적 국가 감시 환경에서는 절대 PWA로 설치하지 마세요. 홈화면의 아이콘 자체가 압수 시 증거물이 됩니다. 브라우저 탭으로 직접 열어 사용하고 닫은 후 기록을 지우거나, 로컬 파일에서 직접 로드하세요.

계산기 위장은 실행 중인 앱을 숨길 뿐 아이콘 자체는 숨기지 못합니다.

💾 저장 형식 옵션에 대하여 (V3.0)

이전 버전에서는 파일 헤더에 무작위 패딩을 주입하여 녹음 파일이 깨졌습니다. V3.0은 깔끔한 두 가지 옵션 모델로 이를 수정했습니다.

  • 그대로 (기본): 녹음 파일을 변경 없이 저장합니다. 어떤 오디오 플레이어로든 바로 재생됩니다. 단순히 증거를 녹음하려는 경우 이 옵션을 사용하세요.
  • 이미지 위장: 녹음 파일을 JPEG 봉투 안에 감싸서 .jpg 확장자로 저장합니다. 갤러리 앱은 이를 "깨진 사진"으로 처리하고 건너뜁니다. 오디오를 재생하려면 컴퓨터에서 IMG_20260105_xxx.jpg를 직접 .webm으로 이름을 바꾸어야 합니다. 봉투 안의 오디오 데이터 자체는 변형 없이 보존되므로, 이름만 바꾸면 완전 재생됩니다.

이 옵션은 언제든 변경할 수 있습니다. 이미 저장된 파일에는 영향을 주지 않으며, 향후 녹음에만 적용됩니다.

두 가지 화면 모델

  • 녹음기 화면: 일반 UX. 토스트·다이얼로그·도움말 버튼이 정상 작동. 첫 실행 시 기본.
  • 계산기 화면: 작동하는 공학용 계산기. 앱은 이 화면에서 절대 침묵 — 토스트·다이얼로그·녹음 인디케이터·도움말 모두 금지. 녹음 중이라도 외관상 그냥 계산기. 위험 환경의 운영 모드.

녹음기 화면 상단의 토글로 전환합니다.

설정 아이콘 숨김

⚙ 아이콘을 숨길 수 있습니다. 앱은 외관상 계산기로만 보입니다. 설정창 복구: 계산기 화면에서 0000(변경한 경우 변경한 코드)을 누른 뒤 =를 누르세요. 코드는 변경 가능합니다.

녹음기 화면에서 "숨기기"를 누르면, 앱이 먼저 계산기 화면으로 자동 전환된 다음 톱니바퀴를 숨기므로 키패드로 복구할 수 있습니다.

왜 녹음 시 비밀번호가 없는가?

이전 버전은 매 녹음 시 비밀번호를 요구했습니다. 현장의 가장 기본 현실에 어긋남이 드러났습니다 — 긴급 상황의 1초 1초가 위험을 키웁니다. V3.0은 비밀번호 장벽을 제거했습니다.

단일 HTML, 외부 의존성 0개

VitalGuard 핵심 철학: 감사 가능한 단일 파일 = 공급망 공격 면역. 외부 라이브러리·CDN·API 모두 0개. SHA-256 해시 1개로 검증.

음향·키워드·BLE 트리거

총성·폭발음·비명은 100~120dB의 명확한 진폭 스파이크를 보입니다. AnalyserNode로 RMS와 Zero-Crossing Rate를 분석합니다. 외부 ML 라이브러리 없이 순수 DSP.

계산기 화면에서 트리거 발동 시 녹음은 완전 침묵으로 시작됩니다 — 토스트 없음, 인디케이터 없음.

패닉 와이프 (3-탭)

"즉시 삭제" 버튼은 검사관에게 단서를 줍니다. 좌상단 보이지 않는 영역을 1초 이내 3번 탭하면 모든 녹음·설정·캐시가 즉시 영구 삭제. 외관상 변화 없음. 두 화면 모두에서 작동.
